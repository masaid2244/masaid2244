#BUG-0247 ⬤ Critical Race condition: duplicate exam submission under high concurrency ✓ Resolved
Steps to Reproduce
  1. 1Open exam in 2 browser tabs simultaneously (same session)
  2. 2Complete all questions in Tab 1, click Submit
  3. 3Within 500ms, click Submit in Tab 2 (before redirect)
  4. 4Observe database records & candidate score report
Expected Result
System accepts only the first submission. Second request returns HTTP 409 Conflict. Candidate score is recorded once. No duplicate entry in exam_submissions table.
Actual Result
Both submissions accepted — score recorded twice. Final score reflects second (incomplete) submission. Candidate fails despite completing exam. Data integrity violation in production DB.
